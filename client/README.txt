====================================================================
  GemStone IV Private Server — Client HUD
  Setup & Usage Guide
====================================================================

FOLDER STRUCTURE
────────────────
Drop these files so your layout looks like this:

  your_server_root\
  │
  ├── server\                  ← your existing server code
  │   └── core\
  │       └── session.py       ← REPLACE with session_patched.py
  │
  ├── scripts\                 ← your existing Lua zone/room files
  │   └── zones\
  │       └── ...
  │
  └── client\                  ← NEW — everything below goes here
      ├── gsiv_hud.py          ← the HUD client
      ├── map_builder.py       ← run once to build the room graph
      ├── session_patched.py   ← (just for reference — copy to server/core/session.py)
      └── data\
          └── room_graph.json  ← AUTO-GENERATED by map_builder.py


────────────────────────────────────────────────────────────────────
STEP 1 — Apply the server patch (ONE FILE CHANGE)
────────────────────────────────────────────────────────────────────
Replace your server's session.py with the patched version.

From your server root directory:

  copy client\session_patched.py server\core\session.py

That's the only server-side change. It adds a single status line
before every > prompt so the HUD can read HP/MP/SP/ST/RT/Room.

The line looks like:
  <GSIV HP=85/100 MP=40/100 SP=10/10 ST=100/100 RT=0 ROOM=100>

It is invisible to the HUD (filtered out before display) but plain
telnet users will just see it scroll past harmlessly.


────────────────────────────────────────────────────────────────────
STEP 2 — Build the room graph (run ONCE, re-run after adding rooms)
────────────────────────────────────────────────────────────────────
From your server root directory:

  python client\map_builder.py

This scans all scripts\zones\*\rooms\*.lua files and writes:
  client\data\room_graph.json

If your scripts folder is somewhere else:
  python client\map_builder.py --scripts C:\path\to\scripts

Expected output:
  [map_builder] Scanning: ...
  [map_builder] Parsed 635 rooms (1 skipped) across 17 zones.
  [map_builder] Written to: client\data\room_graph.json


────────────────────────────────────────────────────────────────────
STEP 3 — Launch the HUD client
────────────────────────────────────────────────────────────────────
Make sure your game server is running first, then:

  python client\gsiv_hud.py

Connect to a remote server:
  python client\gsiv_hud.py --host 192.168.1.10 --port 4901


────────────────────────────────────────────────────────────────────
USING THE HUD
────────────────────────────────────────────────────────────────────

TEXT PANEL (left side)
  Full ANSI-colored game output, just like a telnet terminal.
  Room titles in cyan, creatures in red, items in yellow, etc.

STATUS BARS (top-right)
  HP / MP / SP / ST bars auto-update after every command.
  Bars turn yellow at 50%, red at 25%.
  Roundtime (RT) shows in yellow next to the input box.

MAP PANEL (middle-right)
  Shows your current room (blue highlight) plus 2 rings of
  neighbors laid out in compass directions (N/S/E/W/NE/etc.).
  Green dashed lines = up/down/out exits.
  Hover a room to highlight it, CLICK to pathfind.

PATHFINDING (:go2 style)
  Click any room on the map → the HUD computes the BFS shortest
  path and walks you there one step at a time, waiting for
  each > prompt before sending the next direction.
  Type any command OR click ✕ Cancel Path to abort.
  Works for rooms up to hundreds of steps away.

COMMAND INPUT (bottom)
  Type commands and press Enter.
  Up/Down arrows scroll command history.
  Escape clears the input box.

ROOM INFO PANEL (bottom-right)
  Room name, room number (#ID), and zone displayed live.


────────────────────────────────────────────────────────────────────
REQUIREMENTS
────────────────────────────────────────────────────────────────────
  • Python 3.10 or newer  (https://python.org)
  • tkinter — included with standard Python on Windows
  • No other packages needed

To verify Python is installed:
  python --version

To verify tkinter works:
  python -c "import tkinter; print('tkinter OK')"


────────────────────────────────────────────────────────────────────
TROUBLESHOOTING
────────────────────────────────────────────────────────────────────

"No room data / Run map_builder.py"
  → You haven't run map_builder.py yet, or it ran with the wrong
    --scripts path. Check that client\data\room_graph.json exists.

Map shows rooms but they're all wrong
  → Re-run map_builder.py after any room file changes.

Status bars never update
  → Make sure session_patched.py is in server\core\session.py
    (not the original). Restart the server after copying.

Can't connect
  → Check that your server is running: it listens on port 4901.
    If on a different machine, use --host <server_ip>.

Pathfinding says "No path found"
  → The target room is in a different zone with no connecting
    exits loaded. The map graph only knows about exits declared
    in the Lua room files.
